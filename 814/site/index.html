    一、程序阅读与理解题

    第 1 题

    阅读下面程序，请写出执行后的输出结果。

    1
2
3
4
5
6
    #include <stdio.h>
int main() {
    int x = 6, y = 7;
    printf("%d %d\n", x++, ++y);
    return 0;
}

    1
    6 8

    函数的参数传递是从右向左传递的，所以语句 printf("%d %d\n", x++, ++y); 的参数传递顺序为：

    1. y 自增 \(1\)，传入变量 y；
    2. 传入 x 的值，x 自增 \(1\)。

    故 printf 的形参分别是 值 6、变量 y，结果输出为 6 8。

    注意，后缀自增运算符是先使用变量的值，再将变量自增，而前缀自增运算符则是先将变量自增，再使用变量本身。

    文件操作常见的函数
    函数 功能
    fopen(filename, mode) 打开文件，返回文件指针
    fclose(fp) 关闭文件

    其中，filename 是文件名，mode 是打开模式，包括：

    字符 含义
    r 只读模式，文件必须存在
    w 只写模式，文件不存在时创建，存在时覆盖
    a 只写模式，文件不存在时创建，存在时在尾部追加写入
    + 与上面的模式组合，表示以读写模式打开文件，既可写入数据，也可读取数据
    b 与上面的模式组合，表示打开二进制文件
    函数 功能
    fgetc(fp) 从文件中读取一个字符，并使文件指针指向下一个字符，若已到文件末尾，则返回 EOF
    fputc(ch, fp) 将字符 ch 写入文件
    feof(fp) 判断文件指针是否已到文件末尾
    fgets(str, n, fp) 从文件中读取一行，存入字符串 str，最多读取 n - 1 个字符，若已到文件末尾或出错，则返回 NULL
    ferror(fp) 判断文件操作是否出错
    fputs(str, fp) 将字符串 str 写入文件
    fscanf(fp, format, ...) 从文件中读取格式化输入
    fprintf(fp, format, ...) 将格式化输出写入文件
    rewind(fp) 将文件指针指向文件开头
    fread(buffer, size, count, fp) 从文件中读取 count 个 size 字节的数据，存入 buffer（一般用于读取结构体）
    fwrite(buffer, size, count, fp) 将 buffer 中的 count 个 size 字节的数据写入文件
    fseek(fp, offset, fromwhere) 将文件指针从 fromwhere 位置移动 offset 字节，offset 为正表示向后移动，为负表示向前移动
    ftell(fp) 返回文件指针当前位置离文件开头的字节数

    第 1 题

    现有一个线性表 \(L = (a, c, a, d, b)\)，求 ListLength(L)、ListEmpty(L)、GetElem(L, 3, e)、LocateElem(L, a)、ListInsert(L, 4, f) 和 ListDelete(L, 3) 等基本运算的执行结果。

    顺序表的存储下标从 \(0\) 开始，但是在程序中，我们习惯从 \(1\) 开始，所以在实现中，我们传递以 \(1\) 为起始的下标，实际操作时，我们将下标减 \(1\)。故答案为：

    运算 结果
    ListLength(L) \(5\)
    ListEmpty(L) false(0)
    GetElem(L, 3, e) e = a
    LocateElem(L, a) \(1\)
    ListInsert(L, 4, f) \(L = (a, c, a, f, d, b)\)
    ListDelete(L, 3) \(L = (a, c, d, b)\)

    int LocateElem(SqList L, ElemType e) {
    int i;
    for (i = 0; i < L.length; i++)
        if (L.elem[i] == e) return i+1;      // 查找成功，返回序号 i+1
    return 0;                                // 查找失败，返回 0
}
    为确定元素在顺序表中的位置，需和给定值进行比较的数据元素个数的期望值称为查找算法在查找成功时的平均查找长度（Average Search Length，ASL）。

    假设 \(p_i\) 是查找第 \(i\) 个元素的概率，\(C_i\) 为找到表中其关键字与给定值相等的第 \(i\) 个记录时，和给定值已进行过比较的关键字个数，则在长度为 \(n\) 的线性表中，查找成功时的平均查找长度为：

    \[ ASL = \sum_{i=1}^{n} p_i C_i \]

    一般情况下，\(C_i=i\)，假设每个元素的概率相等，即：

    \[ p_i = \dfrac{1}{n} \]

    则有：

    \[ ASL = \dfrac{1}{n} \sum_{i=1}^{n} i = \dfrac{n(n+1)}{2} \]

    故顺序表的查找操作的平均时间复杂度为 \(O(n)\)。

    Status ListInsert(SqList &L, int i, ElemType e) {
    if (i < 1 || i > L.length+1) return ERROR;  // i 值不合法
    if (L.length == MAXSIZE) return ERROR;      // 当前存储空间已满
    for (int j = L.length - 1; j >= i - 1; j--) 
        L.elem[j + 1] = L.elem[j];              // 插入位置及以后的元素后移
    L.elem[i - 1] = e;                          // 将新元素 e 放入第 i 个位置
    ++L.length;                                 // 表长加 1
    return OK;
}
    假设 \(p_i\) 是在第 \(i\) 个元素之前插入一个元素的概率，\(E_{\text{ins}}\) 为在长度为 \(n\) 的线性表中插入一个元素时所需移动元素次数的期望值（平均次数），则有：

    \[ E_{\text{ins}} = \sum_{i=1}^{n+1} p_i(n-i+1) \]

    假设每个元素的概率相等，即：

    \[ p_i = \dfrac{1}{n+1} \]

    则有：

    \[ E_{\text{ins}} = \dfrac{1}{n+1} \sum_{i=1}^{n+1} (n-i+1) = \dfrac{n}{2} \]

    故顺序表的插入操作的平均时间复杂度为 \(O(n)\)。

    Status ListDelete(SqList &L, int i) {
    if (i < 1 || i > L.length) return ERROR;  // i 值不合法
    for (int j = i; j <= L.length - 1; j++) 
        L.elem[j - 1] = L.elem[j];            // 被删除元素之后的元素前移
    --L.length;                               // 表长减 1
    return OK;
}
    假设 \(p_i\) 是删除第 \(i\) 个元素的概率，\(E_{\text{del}}\) 为在长度为 \(n\) 的线性表中删除一个元素时所需移动元素次数的期望值（平均次数），则有：

    \[ E_{\text{del}} = \sum_{i=1}^{n} p_i(n-i) \]

    假设每个元素的概率相等，即：

    \[ p_i = \dfrac{1}{n} \]

    则有：

    \[ E_{\text{del}} = \dfrac{1}{n} \sum_{i=1}^{n} (n-i) = \dfrac{n-1}{2} \]

    故顺序表的删除操作的平均时间复杂度为 \(O(n)\)。

    int Search_Bin(SSTable ST, KeyType key) {
    int low = 1, high = ST.length;                        // 置查找区间初值
    while (low <= high) {
        int mid = (low + high) / 2;
        if (ST.R[mid].key == key) return mid;            // 找到待查元素
        else if (key < ST.R[mid].key) high = mid - 1;    // 继续在前一子表进行查找
        else low = mid + 1;                              // 继续在后一子表进行查找
    }
    return 0;                                            // 表中不存在待查元素
}

    借助于决策树，很容易求得折半查找的平均查找长度。为讨论方便起见，假定有序表的长度 \(n=2^k-1\)，则决策树是深度为 \(h = \log_2(n+1)\) 的满二叉树。树中层次为 \(1\) 的节点有 \(1\) 个，层次为 \(2\) 的节点有 \(2\) 个，依次类推，层次为 \(h\) 的节点有 \(2^h - 1\) 个。假设表中每个记录的查找概率相等 \(P_i=\dfrac 1 n\)，则查找成功时折半查找的平均查找长度为：

    \[ \begin{aligned} ASL &= \sum_{i=1}^{n} P_i C_i \\ &= \dfrac 1 n \sum_{j=1}^{n} j\cdot 2^{j-1} \\ &= \dfrac{n+1}{n}\log_2(n+1)-1 \\ \end{aligned} \]

    当 \(n\) 较大时，\(ASL\) 近似为 \(\log_2(n+1) - 1\)。因此，折半查找的平均时间复杂度为 \(O(\log_2 n)\)。
